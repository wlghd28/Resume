#### 3.1.2. Dithering
https://github.com/wlghd28/Dithering

디더링을 경험하며 개인적으로 테스트하기 위해 개발했던 프로젝트입니다.
해당 프로젝트는 제가 개발자로서 처음으로 몰두하게 만들었습니다. 인턴 때 처음 부여받은 프로젝트가 회사에서 사용하는 기존 알고리즘(Floyd Steinberg)의 단점을 극복하기 위해서 DBS(Direct Binary Search) 알고리즘의 구현하는 것이었습니다. 인터넷 검색을 해도 자료가 많이 나오진 않아서 꽤나 고생했었습니다. 

프로젝트를 완료하기 위해 노력했던 과정을 간단하게 정리하면
1. 논문을 분석하면서 나왔던 개념들 인터넷 검색하며 정리
2. 논문에 나온 알고리즘을 이해한 그대로 코드로 구현해보기
3. 논문 저자의 정보를 찾아보기
4. 논문 저자의 공개된 개발 자료들이 있는지 찾아보기
   
이 과정을 무려 42일 동안 한 결과 결국 DBS에 대한 알고리즘이 Matlab 코드로 구현되어 있는 소스를 발견했고 오픈소스 이기에 바로 C언어로 재구현에 들어갔습니다. DBS 알고리즘은 연산하기 전 준비과정(가우시안필터, 컨볼루션 연산 등)이 필요한데 그에 대한 계산은 Matlab에선 단 한줄 함수로 구현되어 있었습니다. 전 C언어로 재구현을 해야 했기에 준비과정 연산에 대해서 찾아보고 그 연산을 직접 구현했습니다.    
디더링 프로젝트는 제가 개발하면서 굉장히 어려운 문제를 만났을 때 극복해 나아갈 수 있는 끈기 및 자신감이 크게 생긴 경험이라 기억에 오래 남네요.

라이브러리를 통해 읽어들인 이미지 데이터를 화면에 실시간으로 뿌림과 동시에 이미지의 줌 인, 줌 아웃을 구현 하는게 어려웠습니다. 처음에 이미지를 확대 시킬 시 그대로 데이터를 업스케일링 하는 식으로 구현했더니 특정 수치가 넘어가도록 확대 시키면 메모리 초과 및 속도면에서 심각하게 성능이 내려가는 문제가 있었습니다.   
이 문제를 극복하면서 논리적인 사고력이 한층 발전했던게 기억에 남네요. 극복했던 방법으로는 전체 이미지를 업스케일링 하는 방법에서 화면에 보여지는 부분만 확대를 시키는 방법으로 수정했습니다. 즉, 이미지를 확대 시킬수록 사용자 입장에선 이미지가 커지는 것처럼 보이지만 실제 메모리 상으로는 프로그램에 표시되는 크기만큼만 고정적으로 사용되는 것입니다. 이 방법을 사용한 결과 속도면에서 성능이 매우 올라갔고, 메모리 관리 또한 효과적으로 가능했었습니다.

#### 4.1.3. Voltage Data Receiver  
https://github.com/wlghd28/Voltage_Data_Receiver   

LTC6806 Fuel Cell Monitor 로부터 데이터를 읽어오는 프로그램

- 개발 기여도 : 80%
 - 개발 기간 : 2024.06 ~ 2024.07 (1개월)
 - 작성 언어
    - C
 - 설명
    - MCU 펌웨어 (STM32)
    - LTC6806 (Fuel Cell Monitor) 사용
    - SPI 통신 사용  

해당 프로젝트를 하면서 기억에 남았던 것은 비트연산 입니다. LTC6806 모듈과 SPI 통신을 하기 위해서는 명령체계가 비트연산으로 되어있었기 때문에 2바이트 메모리에 비트연산을 했었습니다. 비트연산을 할 때 비트 연산자를 활용하면 속도가 매우 빠르지만 비트필드를 사용하면 속도가 이론상으로 알던 만큼 빠르지 않다는 것을 알게 됐습니다.  
1. 비트연산은 연산자를 활용하여 사용할 것.
2. 비트필드는 메모리의 절약을 위해 사용하는 것이 아닌 이상 속도면에서는 유리한 것이 없다.
